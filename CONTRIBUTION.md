# Contribution Guide

Thanks for contributing to Bible SG.

## How to contribute

1. Fork the repository
2. Create a feature branch
3. Make focused changes
4. Add or update tests when relevant
5. Open a pull request with a clear description

## Development notes

- Keep code simple and readable
- Prefer small, incremental PRs
- Follow existing TypeScript/Next.js patterns
- Update docs in `./docs` when behavior changes

## Commit style

Use clear commit messages, for example:

- `feat(search): add metadata filter`
- `fix(pipeline): handle interrupted batch resume`
- `docs(roadmap): update phase status`

## Reporting issues

When opening an issue, include:

- Steps to reproduce
- Expected behavior
- Actual behavior
- Logs or screenshots if available
- Environment details (OS, Node version, DB setup)
